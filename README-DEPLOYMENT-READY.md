## Deployment in 3 Steps

### 1️⃣ Get GitHub Credentials
```
Go to: https://github.com/settings/tokens
Create a Personal Access Token (classic) with "repo" scope
Copy the token
```

### 2️⃣ Run the Script
```bash
# Navigate to the folder with push-to-github.sh, then:
bash push-to-github.sh

# When prompted:
# Username: goldrusher9009-sketch
# Password: [Your Personal Access Token]
```

### 3️⃣ Enable GitHub Pages
```
Go to: https://github.com/goldrusher9009-sketch/aacg-superadmin/settings/pages
Source: Deploy from a branch
Branch: main
Folder: / (root)
Save
```

**That's it!** Your portal will be live within 60 seconds at:  
`https://goldrusher9009-sketch.github.io/aacg-superadmin/`

## What the Script Does

The `push-to-github.sh` script automatically handles:

1. **Repository Setup**
   - Initializes a new git repository
   - Configures git with proper user settings

2. **File Creation**
   - Creates comprehensive README.md
   - Creates index.html (your portal)

3. **Git Operations**
   - Stages both files
   - Creates an initial commit
   - Adds GitHub as remote
   - Pushes to main branch

4. **Guided Next Steps**
   - Provides GitHub Pages setup instructions
   - Reminds you about credential configuration
   - Clarifies security considerations

**All you need to provide:** GitHub authentication when prompted

## Configuring Supabase Credentials

After deployment, set your credentials in the browser:

1. Visit your portal

3. Run these commands:
```javascript
localStorage.setItem('SB_URL', 'https://your-project.supabase.co');
localStorage.setItem('SB_KEY', 'your-anon-key-here');
```
4. Refresh the page

## System Requirements

To run the deployment script, you need:

- **Git** installed on your computer
  - macOS: Usually pre-installed, or `brew install git`
  - Linux: `apt install git` or equivalent
  - Windows: Download from https://git-scm.com/download/win

- **Bash or equivalent**
  - macOS/Linux: Built-in
  - Windows: Use Git Bash (included with Git for Windows)

- **GitHub Personal Access Token**
  - Create at: https://github.com/settings/tokens
  - Scope: "repo" (full control of private repositories)

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `bash: command not found` | Install Git for Windows from https://git-scm.com/download/win |
| `Permission denied` | Run `chmod +x push-to-github.sh` first |
| Authentication failed | Create a new Personal Access Token with "repo" scope |
| Portal doesn't load | Verify GitHub Pages is enabled in Settings → Pages |
| "Credentials not configured" | Set Supabase credentials in browser console (see above) |

For detailed troubleshooting, see **DEPLOYMENT-INSTRUCTIONS.md**
